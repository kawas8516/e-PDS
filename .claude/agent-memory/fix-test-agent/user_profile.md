## Role
Developer working on e-PDS — a Java EE Dynamic Web Project (Eclipse + Tomcat 10.1, no Maven).

## Project Context
Building/maintaining a government digital ration card management system with citizen and admin roles. Stack is deliberately minimal: no Spring, no Hibernate, no build tools beyond Eclipse WTP.

## Collaboration Style
- Provides structured audit findings (Critical/Important/Optional) for the agent to implement
- Prefers a concise change log (files modified + one-line rationale) rather than full diffs
- Excludes specific items from scope explicitly (e.g., "leave db.properties alone")
- Expects dependency-ordered execution of fixes, not just sequential numbering
