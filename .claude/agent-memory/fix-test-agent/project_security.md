## CSRF Tokens
- Generated by `CSRFUtil.generateToken()` (UUID)
- Validated by `CSRFUtil.validateToken(sessionToken, requestToken)`
- Session attribute name: `csrfToken`
- Seeded in session on every GET that renders a form (LoginServlet, RegisterServlet, ResetPasswordServlet)
- Regenerated in session after every failed POST (error path) so retry forms have a valid token
- Validated at the top of every POST handler before any other processing
- Embedded in forms as `<input type="hidden" name="csrfToken" value="${sessionScope.csrfToken}">`

## AuthFilter
- `@WebFilter(urlPatterns="/*", dispatcherTypes={DispatcherType.REQUEST})` — REQUEST dispatch only
- Public allow-list (exact match or prefix): `/`, `/LoginServlet`, `/RegisterServlet`, `/ResetPasswordServlet`, `/css/`, `/js/`, `/images/`, `/assets/`, `/favicon`
- Unauthenticated access → redirect to `/LoginServlet`
- Unauthorized role access → `sendError(403)` (not redirect — avoids loop)
- RBAC uses `startsWith` prefix checks and explicit path lists, never `contains()`

## RBAC Roles
- Stored in DB as uppercase: `ADMIN`, `CITIZEN` (changed from lowercase `citizen` on 2026-04-28)
- `User.getRole()` returns whatever is in DB; all checks use `.toUpperCase()` or `equalsIgnoreCase()`
- Admin paths: `/admin-dashboard.jsp`, `/admin/`, `/stock.html`, `/approvals.html`, `/reports.html`
- Citizen paths: `/citizen-dashboard.jsp`, `/citizen/`, `/family.html`, `/complaints.html`, `/allocation.html`
- `AuthService.getDashboardPath()` uses `equalsIgnoreCase("ADMIN")` — safe for legacy lowercase data

## Session Management
- `request.changeSessionId()` used after successful authentication (session-fixation prevention)
- Session timeout: 30 minutes (`SESSION_TIMEOUT_SECONDS = 1800`)
- Session cookie: `HttpOnly=true`, `Secure=true`, `SameSite` via Tomcat config
- Logout: POST-only (`/LogoutServlet`), validates CSRF token, then `session.invalidate()`

## Audit Logging
- `AuditUtil.logAction(int userId, String action, String ip)` → inserts into `audit_logs` table
- Actions used: `LOGIN_SUCCESS` (real userId), `LOGIN_FAIL` (userId=0 acceptable), `PASSWORD_RESET` (real userId)
- Password reset audit: user resolved via `UserDAO.findByIdentifier()` before logging

## Password Reset Security
- Unauthenticated reset is gated by requiring current password verification before accepting new password
- Uses `UserDAO.authenticate(username, currentPassword)` to verify current password
- TODO (long-term): replace with email-token flow

## Error Pages
- All 403/404/500 → `/WEB-INF/views/error.jsp`
- `isErrorPage="true"` on error.jsp; reads `jakarta.servlet.error.status_code` and `jakarta.servlet.error.message`
